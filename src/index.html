AI Messenger User Data

Local-first data behavior

  • AI Messenger has no built-in telemetry, analytics, advertising, developer reporting, or remote user account.
  • Chat messages, attachments, profiles, settings, and history remain on the user's device.
  • Prompts and explicitly attached images/files are sent to the local model endpoint configured by the user.

AI Messenger User Guide

Getting started

  1. Start Ollama, then open Tools → Setup local model. Use automatic detection or enter the local server address and choose Save & Connect.
  2. Open Help → Installed Model Guide to see each installed model's size and capabilities.
  3. Click a contact to open a chat. In a one-to-one chat, use the model picker to choose the local model.
  4. Use Tools → About Me to set your name, picture, background, goals, work style, and guidance preferences.
  5. Use the AI profile menu or right-click a contact to set its name, picture, tone, gender, Soul, personality, and skills.
  6. In the main contact list, figurine markers identify capabilities: glasses for OCR or vision, a paintbrush for image generation, and a lightbulb for reasoning models. The timestamp shows the most recent message; after a day passes it includes the date and time.
  7. Use Search active chats in the main toolbar to find matching message text in the latest direct and group conversations. Select a result to open that chat.
  8. Type a message and press Send or Enter. Your message appears immediately; Stop ends active generations. Thinking controls Fast or Deep response effort where supported.
  9. Use Send Files for images or text files, and Share view for a user-selected desktop or window snapshot. Only explicitly attached content is sent to the local model.
  10. Use Invite to create a conversation with multiple assistants. In a one-to-one chat, Invite creates a new group. In an existing group, Invite adds or removes assistants from that same group and retains its name and saved history; messages from removed assistants remain visible with gray names. In a team that includes Chief of Staff, address the Chief with the desired outcome, such as “interpret this poem,” and attach the source. Chief of Staff identifies the right specialists, delegates the work in the shared thread, and synthesizes their real replies. You do not need to micromanage each assistant.
  11. Use the chevron to attach the chat to the contact list. Use Detach to give it its own window; other model tasks continue in the background.
  12. File → New Chat starts a fresh visible session while saved history remains available through Chat History. Loading old history can add outdated context.
  13. Unused assistants suspend after five minutes and show a clock icon; after ten minutes they move Offline. Click the assistant to wake it. Right-click an assistant to suspend or wake it manually.
  14. Use Actions for light/dark mode and Tools for font size, setup, and assistant controls. Help contains the model guide, features, setup guide, license, terms, and user-data information.
  15. To permanently remove AI Messenger data on Windows, quit the app, open %APPDATA%\Retro Messenger Assistant in File Explorer, and delete that folder. This removes the encrypted vault, chats, profiles, and settings.

AI Messenger Setup Guide

Welcome to AI Messenger

  1. Start your local model server.For Ollama, install it, run it, and download at least one chat model. Other local apps must expose an OpenAI-compatible /v1 API.
  2. Open Tools → Setup local model.Choose Detect Ollama automatically or enter the server URL manually. The Ollama default is http://127.0.0.1:11434; keep it bound to localhost.
  3. Match the model to your hardware.Use the Installed Model Guide for size and capability information. Larger models require more RAM or VRAM and take longer to load.
  4. Test the connection.Use Test Connection before saving. Then choose a discovered model from the Model selector at the top of the chat.
  5. Customize each assistant.Use the AI picture arrow for that model’s name and picture. Tone, language gender, Soul, personality, custom instructions, and skills are also saved to the selected model only.
  6. Use visual assistance deliberately.Activities captures a selected screen/window snapshot. Send Picture attaches an image. Images go only to the configured local endpoint.
  7. Manage context.File → New Chat starts clean. Chat History can restore text context, but historical information may be outdated.

Privacy: AI Messenger restricts model connections to this device. Chat state, preferences, configuration, and optional server keys are protected by your standalone vault passphrase.
